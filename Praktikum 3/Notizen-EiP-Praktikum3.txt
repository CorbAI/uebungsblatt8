Praktikum 3: Grundkonzepte der objekt-orientierten Programmierung I: Klasse, Objekt und Methode

Vektor1: Funktionale Prozedur zur Multiplikation zweier Vektoren (imperativer Ansatz)
Vektor2: Objektvektoren, nicht-funktionale und funktionale Methoden (objektorientierter Ansatz)
GetterSetter/EineKlasse
IdentitaetvonObjekten - Objekte haben eine Identität, "Werte" nicht

Beispiel Objektorientierte Programmierung:
Person, Account und BankApplication

Konzepte:
- Objekt und Klasse als Objekt-Schablone
- Prozedur und Methode im Vergleich
- Nicht-funktionale und funktionale Methode
- Rückgabewert (einer funktionalen Prozedur oder einer funktionalen Methode)
- Klassenvariable (oder Klassenattribute) und Klassenmethoden 
- Objektvariable (oder Instanzvariable oder Objektfeld) und Objektmethode (Vorsicht: Ein Objekt-Feld/Instanzvariable ist kein Array!)
- Objekt-Initialisierung (Konstruktor und Initialisierung von Instanzvariablen)
- Einkapselung: Getter und Setter
- Naming convention
- Getter/Setter für Objekte (außer Zeichenketten oder immutable Objekte): ggf. Kopien ("clones") verwenden
- Sichtbarkeiten

public class Example {
  private static int count = 0;
  public Example() { count++; }
  public static void main(String[] args) {
    System.out.println("Anzahl Instanzen: " + count); // 0
    new Example();
    System.out.println("Anzahl Instanzen: " + count); // 1
    new Example();
    System.out.println("Anzahl Instanzen: " + count); // 2
  }
}
